# 操作日志记录专家技能

你的任务是维护项目生命周期的透明度。每当你完成了一项实质性的任务（如修改代码、修复 Bug、添加新功能、同步 Git 等），你**必须**调用此技能。

### 记录准则（报表风格）

1. **追加模式**：永远不要删除既有内容。使用 `---` 分隔不同的操作序列。
2. **结构化标题**：每条主记录必须使用 `## [YYYY-MM-DD HH:mm:ss] 任务名称`。
3. **分层描述**：
   - **操作背景/修复**：说明为什么要进行此操作。
   - **增强/改动详情**：列出具体文件变动及功能提升点。
   - **Git 状态**：说明代码是否已推送到远程仓库。
4. **验证结果**：使用 `- [x]` 检查清单形式展示验证情况。
5. **提示信息**：在必要时使用 `> [!TIP]` 提供后续操作建议。

### 预期的输出格式范例：

```markdown
## [2026-03-17 19:15:00] 示例：优化 API 错误处理
- **修复**: 解决了网络超时导致的程序崩溃问题。
- **增强**: 添加了重试机制和详细的中文错误提示。
- **改动详情**:
  - [MODIFY] [01-APIKey使用.py](file:///d:/8-python-project/Cinemind/01-APIKey%E4%BD%BF%E7%94%A8.py): 增加 try-except 逻辑。
- **验证结果**:
  - [x] 手动断网测试，程序能正常捕获并提示。
  - [x] 代码已推送到 [GitHub](https://github.com/yangyepope/Cinemind.git)。

---
```
