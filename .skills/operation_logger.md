# 操作日志记录专家技能

你的任务是维护项目生命周期的透明度。每当你完成了一项实质性的任务（如修改代码、修复 Bug、添加新功能、同步 Git 等），你**必须**调用此技能。

### 记录准则

1. **追加模式**：永远不要删除 `walkthrough.md` 的既有内容。使用追加方式在文件末尾（或通过逻辑分段）添加新记录。
2. **时间戳**：每条记录必须以本地时间开头，格式建议为 `YYYY-MM-DD HH:mm:ss`。
3. **内容结构**：
   - **操作标题**：简练概括做了什么。
   - **详细描述**：具体修改了哪些文件、使用了什么命令、解决了什么问题。
   - **影响/结果**：操作后的状态（如：测试通过、代码已推送到 origin）。
4. **可视化（可选）**：如果操作涉及 UI 变更，应附带截图或录屏路径。

### 预期的输出格式范例：

```markdown
## [2026-03-17 19:10:00] 初始化 UV 环境
- **操作**: 运行 `uv init` 并添加 `openai` 依赖。
- **改动**: 
  - [NEW] `pyproject.toml`
  - [NEW] `uv.lock`
- **状态**: 成功。本地 Python 环境已切换为 uv 管理，并与远程仓库同步。
---
```
