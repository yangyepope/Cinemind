# 通用编程卓越准则

本技能定义了跨项目的全局开发标准，旨在编写高性能、高安全性和完全合规的代码。

## 1. 性能优先 (Performance)
- **批量操作**：在涉及磁盘、数据库或网络 I/O 时，始终优先采用“批量（Batch）”处理，而非循环执行单次操作。
- **缓存机制**：对于高频访问且极少变动的数据，应引入合理的缓存策略。
- **资源释放**：使用 `with` 语句或类似的上下文管理器，确保文件句柄、网络连接等资源被及时释放。

## 2. 安全加固 (Security)
- **输入校验**：遵循“永不信任用户输入”原则。对于所有来自外部的参数（如 ID、文件名、表单数据），必须进行严格的合法性校验（如类型检查、正则过滤）。
- **防止注入**：构建文件路径、SQL 查询或系统指令时，禁止使用字符串拼接，应使用参数化查询或官方提供的安全拼接函数。
- **敏感信息**：严禁在代码中硬编码任何 API Key、密码或私钥，统一使用环境变量或安全配置文件。

## 3. 合规与健壮性 (Compliance & Robustness)
- **数据一致性**：使用 Pydantic 或类似的模式模型来强制执行数据结构，确保数据在存储和传输过程中的完整性。
- **优雅降级**：当非核心功能（如日志、非关键统计、历史记录）失败时，应捕获异常并确保主流程能继续运行。
- **清晰语义**：变量名应具有“自解释性”，复杂逻辑必须包含中文注释。

## 4. 实施清单 (Action Checklist)
- [ ] 所有的 I/O 是否都采用了最有效的批量方式？
- [ ] 外部输入是否已经过了 `isalnum()` 或正则校验？
- [ ] 异常捕获是否精确且提供了清晰的中文错误提示？
- [ ] 是否遵循了项目的 `project_rules.md` 语言规范？
