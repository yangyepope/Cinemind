本 Skill 指导创建独特的、生产级的辅助界面，避免通用的“AI 废料 (AI slop)”美学。通过对美学细节和创意选择的极大关注，实现真正的、可工作的代码。

用户提供前端需求：要构建的组件、页面、应用程序或界面。他们可能包含关于目的、受众或技术约束的上下文信息。

## 设计思维 (Design Thinking)

在编码之前，请理解背景并致力于一个**大胆的 (BOLD)** 美学方向：
- **目的**：该界面解决什么问题？谁在使用它？
- **基调 (Tone)**：选择一个极致的方向：极简主义 (brutally minimal)、极致主义混沌 (maximalist chaos)、复古未来主义 (retro-futuristic)、有机/自然 (organic/natural)、豪华/精致 (luxury/refined)、俏皮/玩具感 (playful/toy-like)、社论/杂志感 (editorial/magazine)、粗犷/原始 (brutalist/raw)、装饰艺术/几何 (art deco/geometric)、柔和/粉彩 (soft/pastel)、工业/实用主义 (industrial/utilitarian) 等。有如此多的风格可供选择。使用这些作为灵感，但要设计出一个真正符合美学方向的界面。
- **约束**：技术要求（框架、性能、辅助功能）。
- **差异化**：是什么让它**难以忘怀 (UNFORGETTABLE)**？人们会记住哪一点？

**关键**：选择一个明确的概念方向并精准执行。大胆的极致主义和精致的极简主义都有效——关键在于意图，而非强度。

然后实现可工作的代码（HTML/CSS/JS, React, Vue 等），要求：
- 生产级且功能完备
- 视觉震撼且令人难忘
- 具有清晰的美学视角并保持连贯
- 在每一个细节上都经过精心打磨

## 前端美学指南 (Frontend Aesthetics Guidelines)

关注以下方面：
- **排版 (Typography)**：选择美丽、独特且有趣的字体。避免使用 Arial 和 Inter 等通用字体；相反，选择提升前端美学的独特字体；意外的、具有个性的字体选择。将独特的展示字体与精炼的正文字体配对。
- **颜色与主题 (Color & Theme)**：致力于一种连贯的美学。使用 CSS 变量以保持一致性。主色调搭配鲜明的强调色，其效果优于胆怯、平庸的调色板。
- **动效 (Motion)**：使用动画来实现特效和微交互。对于 HTML，优先考虑纯 CSS 方案。在 React 中，如果可用，请使用 Motion 库。专注于高影响力的时刻：一个编排精良、带有交错揭示 (animation-delay) 的页面加载，比零散的微交互更能创造惊喜。使用滚动触发和令人惊喜的悬停状态。
- **空间构图 (Spatial Composition)**：意想不到的布局。不对称。重叠。对角线流动。打破网格的元素。慷慨的负空间或受控的密度。
- **背景与视觉细节 (Backgrounds & Visual Details)**：营造氛围和深度，而不是默认使用纯色。添加与整体美学匹配的上下文特效和纹理。应用创意形式，如渐变网格 (gradient meshes)、躁点纹理 (noise textures)、几何图案、分层透明、强烈的阴影、装饰性边框、自定义光标和颗粒感叠加。

**绝对不要**使用通用的 AI 生成美学，例如过度使用的字体族（Inter, Roboto, Arial, 系统字体）、陈词滥调的配色方案（特别是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏特定语境特征的模板化设计。

发挥创意进行诠释，并做出意想不到的选择，让人感觉是真正为该语境设计的。没有两个设计应该是相同的。在明暗主题、不同字体、不同美学之间切换。**绝对不要**在每一代设计中都汇聚到共同的选择（例如 Space Grotesk）。

**重要**：将实现复杂度与美学愿景相匹配。极致主义设计需要包含大量动画和特效的精细代码。极简主义或精炼的设计需要克制、精准，并细致关注间距、排版和微妙细节。优雅源于对愿景的完美执行。
