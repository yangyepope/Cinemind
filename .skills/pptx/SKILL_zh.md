## 设计理念 (Design Ideas)

**不要创建乏味的幻灯片。** 在白色背景上仅放置平淡的项目符号不会给人留下深刻印象。请考虑为每张幻灯片应用此列表中的想法。

### 在开始之前

- **选择一个大胆的、与内容相符的调色板**：调色板应感觉是为“这个主题”专门设计的。如果将你的颜色应用到完全不同的演示文稿中仍然“行得通”，说明你的选择针对性不够。
- **主导地位优于平等**：一种颜色应占据主导地位（60-70% 的视觉权重），搭配 1-2 种支撑色调和一种鲜明的强调色。绝不要给所有颜色相等的权重。
- **暗/亮对比**：标题页 + 结束页使用深色背景，内容页使用浅色背景（“三明治”结构）。或者致力于贯穿始终的深色背景以获得高端感。
- **致力于一种视觉基调**：选择一个独特的元素并重复使用 —— 圆角图像框、彩色圆圈内的图标、粗的单侧边框。在每张幻灯片中贯彻这一元素。

### 调色板推荐

选择与你的主题匹配的颜色 —— 不要默认使用通用的蓝色。使用这些调色板作为灵感：

| 主题 | 主色 | 次色 | 强调色 |
|-------|---------|-----------|--------|
| **深夜高管 (Midnight Executive)** | `1E2761` (藏青) | `CADCFC` (冰蓝) | `FFFFFF` (白) |
| **森林与苔藓 (Forest & Moss)** | `2C5F2D` (森林绿) | `97BC62` (苔藓绿) | `F5F5F5` (奶油色) |
| **珊瑚能量 (Coral Energy)** | `F96167` (珊瑚红) | `F9E795` (金色) | `2F3C7E` (藏青) |
| **温暖陶土 (Warm Terracotta)** | `B85042` (陶土色) | `E7E8D1` (沙色) | `A7BEAE` (鼠尾草绿) |
| **海洋渐变 (Ocean Gradient)** | `065A82` (深蓝) | `1C7293` (鸭绿) | `21295C` (午夜蓝) |
| **炭黑极简 (Charcoal Minimal)** | `36454F` (炭黑) | `F2F2F2` (米白) | `212121` (黑) |
| **鸭绿信任 (Teal Trust)** | `028090` (鸭绿) | `00A896` (海泡绿) | `02C39A` (薄荷绿) |
| **浆果与奶油 (Berry & Cream)** | `6D2E46` (浆果色) | `A26769` (暗玫瑰红) | `ECE2D0` (奶油色) |
| **舒缓鼠尾草 (Sage Calm)** | `84B59F` (鼠尾草绿) | `69A297` (桉树绿) | `50808E` (石板蓝) |
| **樱桃大胆 (Cherry Bold)** | `990011` (樱桃红) | `FCF6F5` (米白) | `2F3C7E` (藏青) |

### 针对每张幻灯片

**每张幻灯片都需要视觉元素** —— 图像、图表、图标或形状。纯文本幻灯片很容易被遗忘。

**布局选项：**
- 双栏布局（左侧文本，右侧插图）
- 图标 + 文本行（彩色圆圈内的图标、粗标题、下方描述）
- 2x2 或 2x3 网格（一侧为图像，另一侧为内容块网格）
- 半屏图像（全屏占据左侧或右侧）搭配内容叠加

**数据展示：**
- 大号统计数据呼号（60-72pt 的大数字，下方带小标签）
- 对比栏（前后对比、优缺点、侧重选项对比）
- 时间线或流程图（编号步骤、箭头）

**视觉打磨：**
- 在章节标题旁放置彩色小圆圈内的图标
- 关键统计数据或口号使用斜体强调文本

### 排版建议

**选择一个有趣的字体配对** —— 不要默认使用 Arial。选择一个有个性的标题字体，并搭配一个简洁的正文字体。

| 标题字体 | 正文字体 |
|-------------|-----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 元素 | 字号 |
|---------|------|
| 幻灯片标题 | 36-44pt 加粗 |
| 章节标题 | 20-24pt 加粗 |
| 正文文本 | 14-16pt |
| 说明文字 | 10-12pt 柔和色 |

### 间距原则

- 最小 0.5 英寸边距
- 内容块之间留出 0.3-0.5 英寸间隙
- 留出呼吸空间 —— 不要填满每一英寸

### 避免事项 (常见错误)

- **不要重复相同的布局** —— 在不同幻灯片间切换栏位、卡片和呼号布局
- **不要将正文居中** —— 段落和列表请左对齐；仅标题采用居中对齐
- **不要忽略大小对比** —— 标题需要 36pt+ 才能从 14-16pt 的正文中脱颖而出
- **不要默认使用蓝色** —— 选择能反映特定主题的颜色
- **不要随机混合间距** —— 选择 0.3 英寸或 0.5 英寸的间隙并始终保持一致
- **不要仅修饰一张幻灯片而忽略其他** —— 要么全面投入，要么全程保持简洁
- **不要创建纯文本幻灯片** —— 添加图像、图标、图表或视觉元素；避免单纯的标题 + 项目符号
- **不要忘记文本框填充 (Padding)** —— 当线条或形状与文本边缘对齐时，在文本框上设置 `margin: 0` 或偏移形状以补偿填充
- **不要使用低对比度元素** —— 图标和文本都需要与背景有强烈的对比；避免在浅色背景上使用浅色文字或在深色背景上使用深色文字
- **绝对不要在标题下使用强调线** —— 这是 AI 生成幻灯片的典型标志；请改用负空间或背景色

## 质量保证 (QA) —— 必修课

**假设存在问题。你的工作就是找出它们。**

你的第一次渲染几乎绝不是准确无误的。将 QA 视为漏洞搜寻，而非确认步骤。如果你在第一次检查时没有发现任何问题，说明你找得不够仔细。

### 内容 QA

```bash
# 提取文本
python -m markitdown output.pptx
```

检查内容缺失、错别字、顺序错误。

**使用模板时，检查是否残留占位符文本：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

如果 grep 返回结果，请在宣称成功前修复它们。

### 视觉 QA

**⚠️ 使用子智能体 (Subagents)** —— 即使只有 2-3 张幻灯片。你一直盯着代码看，会看到你“预期”的结果，而不是“实际”的结果。子智能体拥有全新的视角。

将幻灯片转换为图像（见[转换为图像](#converting-to-images)），然后使用此提示词：

```
视觉检查这些幻灯片。假设存在问题 —— 请找出它们。

重点查看：
- 元素重叠（文本穿过形状、线条穿过文字、堆叠元素布局错误）
- 文本溢出或在边缘/框边界处被切断
- 装饰线是针对单行文本定位的，但标题换行成了两行
- 来源引用或页脚与上方内容冲突
- 元素过于靠近（间隙 < 0.3 英寸）或卡片/章节几乎相撞
- 间隙不均（一处大块空白，另一处过于拥挤）
- 距离幻灯片边缘的边距不足（< 0.5 英寸）
- 栏目或类似元素对齐不一致
- 低对比度文本（例如：奶油色背景上的淡灰色文字）
- 低对比度图标（例如：深色背景上的深色图标且没有对比圆圈）
- 文本框过窄导致过度换行
- 残留的占位符内容

针对每张幻灯片，列出问题或疑虑，即使是微小的细节。

阅读并分析这些图像：
1. /path/to/slide-01.jpg (预期：[简要描述])
2. /path/to/slide-02.jpg (预期：[简要描述])

报告发现的所有问题，包括细微问题。
```

### 验证循环

1. 生成幻灯片 → 转换为图像 → 检查
2. **列出发现的问题** (如果没有发现，请重新批判性地寻找)
3. 修复问题
4. **重新验证受影响的幻灯片** —— 一个修复往往会产生另一个新问题
5. 重复直到完整检查不再发现新问题

**在完成至少一个“修复-验证”循环之前，不要宣称成功。**

## 转换为图像

将演示文稿转换为单个幻灯片图像以便进行视觉检查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

这将创建 `slide-01.jpg`、`slide-02.jpg` 等文件。

在修复后重新渲染特定幻灯片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```
