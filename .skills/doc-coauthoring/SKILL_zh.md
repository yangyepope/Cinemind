# 协作编写 (Co-authoring)

## 概览

与 Claude 协作编写长文档是一项多阶段、迭代的任务。本 Skill 旨在协助进行上下文收集、通过反复试验（读者测试）进行迭代写作，以及最终交付物的连贯性。

**关键词**: 协作编写、长文档协作、上下文收集、迭代编辑、读者测试、交付物连贯性、内容策略、联合编辑

## 指南

### 阶段 1：理解 (The Deep Dive)（15-20% 的时间）

不要急于动笔。首先要构建该文档存在的“世界视角”。

1.  **收集上下文**：
    - **核心信息**：这是关于什么的？谁是受众？
    - **背景材料**：是否有之前编写的草案、大纲、研究笔记或过往数据可以利用？
    - **目的**：其目标是什么（销售、通知、说服）？
2.  **定义声音与基调**：协作编写需要匹配某种特定风格。
3.  **大纲设计**：在动笔前就文档的结构达成一致。

### 阶段 2：执行 (The Draft)（60-70% 的时间）

本阶段是主要工作发生的地方，通过迭代和协作完成。

1.  **增量写作**：一次专注于一个章节。
2.  **协作反馈循环**：Claude 起草一个章节，由共创者（用户）审查，然后 Claude 根据反馈进行改进。
3.  **保持动态大纲**：随着写作过程中新想法的出现，调整原始大纲。

### 阶段 3：连贯性 (The Final Polish)（10-15% 的时间）

确保所有章节感觉像是出自同一个人之手。

1.  **统一声音**：调整某些章节以匹配整体基调。
2.  **逻辑流与转换**：确保各章节之间的过渡自然。
3.  **最终编辑与校对**：最后一次通读，检查语法、拼写和清晰度。

## 读者测试 (Reader Testing)

在写作过程中使用读者测试。

- **模拟读者反馈**：询问 Claude 如果不同类型的读者（例如：“一位持怀疑态度的 CEO”、“一位好奇的初学者”、“一位时间紧迫的分析师”）阅读此章节会有什么反应。
- **差距分析**：在完成一个章节后，询问：“这个章节缺少了什么来达到其既定目标吗？”

## 技术与工作流提示

- **使用 Markdown**：Markdown 在协作编写中最易于阅读。
- **引用之前的上下文**：定期提及在“理解”阶段确定的目标。
- **接受批评**：Claude 已准备好根据反馈重写每一个词；共创者应感到在此过程中可以提出严厉要求。

## 结论

协作编写是一种伙伴关系。Claude 负责生成内容并遵循指南，而共创者（用户）负责根据独特的洞察力和对最终目标的理解来掌控方向。
