# Implementation Plan: RAG 智能路由优化 (2026-03-24)

本方案旨在修复 RAG 链路过于激进的“防御性分支”，确保通用对话和人设问答能够正常进行。

## 待修复痛点

> [!WARNING]
> 目前当检索库为空时，系统会直接掐断对话。这导致用户问“你是谁”、“你好”这类不需要检索库的问题时，也会收到“无能为力”的生硬回复。

## 优化逻辑

### 路由优先级重构

我们将 `RunnableBranch` 的逻辑调整为以下顺序：

1.  **第一优先级 (元对话识别)**：
    -   条件：如果问题包含“你是谁”、“你好”、“功能”等关键词，或者 `standalone_query` 判定为非知识检索类。
    -   动作：直接进入 LLM，让模型根据 System Prompt 的“星级导购”人设进行回复。
2.  **第二优先级 (知识缺失拦截)**：
    -   条件：如果上面的元对话没命中的，且 `context` 物理检索结果确实为空。
    -   动作：返回“机密库暂无记录”的防御性回复。
3.  **第三优先级 (正常检索流程)**：
    -   条件：默认。
    -   动作：携带 `context` 进入 LLM 推理。

## 验证计划

- **测试用例 1**：输入“你是谁”。预期返回“我是影心 Cinemind 的星级导购...”
- **测试用例 2**：输入无关高难度问题。预期返回防御性拦截提示。
